# TypeScript SDK — Developer Guide

Package: `@together-sandbox/sdk`

## File layout

```
src/
├── TogetherSandbox.ts          # ← hand-written facade (only non-generated file)
├── index.ts                    # Public re-exports
└── api-clients/
    ├── api/                    # Generated — management API client
    └── sandbox/                # Generated — in-VM sandbox API client
```

**Only `TogetherSandbox.ts` and `index.ts` are hand-edited.**
Everything inside `api-clients/` is regenerated by `npm run generate` and must
not be modified directly.

## Facade patterns

### Always unwrap data from responses

Facade methods must return the actual payload, not the raw response object:

```typescript
// ✅ correct
const result = await sandboxApi.readFile({
  client,
  path: { path },
  throwOnError: true,
});
return result.data.content;

// ❌ wrong — returns the full response wrapper
return result;
```

### Always set `throwOnError: true`

Every API call in the facade must include `throwOnError: true`. This converts
HTTP errors into thrown exceptions so callers don't have to check `.error`.

```typescript
// ✅ correct
const result = await api.startSandbox({
  client: this._apiClient,
  path: { id },
  throwOnError: true,
});

// ❌ wrong — error handling is missing
const result = await api.startSandbox({
  client: this._apiClient,
  path: { id },
});
```

### File uploads use binary blobs only

Always convert string content to `Blob` with `application/octet-stream` before
sending to `createFile`. Never send content as a plain string or JSON body.

```typescript
const body =
  typeof content === "string"
    ? new Blob([content], { type: "application/octet-stream" })
    : content;
```

### Namespace delegation pattern

Sub-namespaces (`.files`, `.execs`, etc.) are plain objects returned by getters,
not class instances. The internal `_sandboxClient` and `_apiClient` must remain
private — they are never exposed on the public API surface.

### Section dividers

Use the established ASCII box-drawing divider style for logical sections:

```typescript
// ─── Section Name ────────────────────────────────────────────────────────────
```

## TypeScript conventions

- **`verbatimModuleSyntax` is enabled.** Always use `import type` for
  type-only imports.
- **No `@ts-ignore`.** Use `// @ts-expect-error` with a brief comment
  explaining why the suppression is necessary.
- **Strict mode is on.** Don't disable `strict` or any of its sub-flags.

## Build and test

```bash
npm run generate   # regenerate api-clients/ from OpenAPI specs
npm run build      # compile with tsup (CJS + ESM output)
npm test           # run vitest unit tests
```

## Testing conventions

- Tests live alongside source in `src/TogetherSandbox.test.ts`.
- Use `vitest` (`describe` / `it` / `expect`).
- Use `makeVmInfo()` helper factories to build minimal valid fixture objects.
- Test that internal clients are **not** exposed as public properties.
- Test that renamed/removed methods are absent from the public namespaces.
